校园网多设备共享上网与openwrt的Drcom web认证总结

适用openwrt，DrCOM

前言

在大二上时期也就是2025年下半年学校放弃了宽带，全面转变成WIFI,故方案采取的是无线中继 + web认证方式

校园网绕过多设备限制原理

绕过多设备上网检测原理不在赘述,会在本小节结尾贴出对应的网址。
分别有UA3F,UA2F,UA-MASK可以绕过。笔者用的是UA3F。(会在本小节结尾贴出各自的github地址与UA3F官方教程)
UA3F的官方教程已经很简单了,故不会重复说明.请参考官方教程。

Drcom web认证

无线中继(有线可以跳过)

点击网络——无线,找到并点击扫描 然后挑一个信号强的点击加入网络 然后点击提交 模式为客户端(cilent)即可 这时候路由器应该是连接上校园网了。

Drcom web认证

Web认证有两种方式，一种是前文1v7w大佬帖子(帖子链接:https://www.cnblogs.com/1v7w/p/17642767.html)里的shell脚本,第二种则是1v7w大佬做的的openwrt插件。

配置获取

在电脑上打开未认证时的登录网页，然后按F12(开发者模式)，打开网络(Network)选项，接着认证登录，点击其中几个包，找到形似截图的这个包. 点击带有login?callback…….的包 看到请求网址那里，需要记录下：
请求网址开头的ip，记为server。
server冒号后面的数字，记为port
wlan_ac_name=后面的字符串，直到&为止(不包括&下同)
wlan_user_ip=后面的ip，这是学校分配下的内网ip
远程地址(我的是172.22.44.46:801),这是校园网服务器的IP(把:与它后面的数字除去) (如果觉得麻烦可以交给AI来做)

DrCOM登录脚本

贴出原作者的shell脚本,将以下配置换成自己的，然后上传至路由器的/etc/storage目录下,然后添加crontab计划任务,给予文件执行权限，如果没有/etc/storage目录，就创建 执行权限获取
获取执行权限

1

chmod +x /etc/storage/drcom_cron.sh

创建目录

1

mkdir /etc/storage/

#!/bin/bash
# 西安石油大学 校园网登录脚本 2023.08
# 适用于: openwrt
# by: 1v7w

# 本脚本保存至 /etc/storage/drcom_cron.sh

# 添加计划任务
#『系统 - 计划任务』
# */10 * * * * /etc/storage/drcom_cron.sh

# --------- 配置区 ---------
server=10.123.0.253     # 修改成你校校园服务器IP
user=202000000000		# 修改成你的账号
password="password"		# 修改成你的密码
wlanacname="HUXIAN_C1_1F_107_H3C_AC_01"	# 修改成你所连接的AP的名字
# --------- 配置区 ---------

html_file="/tmp/drcom_html.log"
login_out_file="/tmp/drcom_login.log"

logger -t "【Dr.COM网页认证】" "开始定时检测"
curl -s "http://${server}" > ${html_file}
check_web=`grep "Dr.COMWeb" ${html_file} | head -n1`
check_status=`grep "Dr.COMWebLoginID_0.htm" ${html_file} | head -n1`
# Dr.COMWebLoginID_0.htm 登录页（未登录）
# Dr.COMWebLoginID_1.htm 注销页（已登录）
# Dr.COMWebLoginID_2.htm 登录失败页
# Dr.COMWebLoginID_3.htm 登录成功页

if [[ "$check_web" == "" ]]; then
  logger -t "【Dr.COM网页认证】" "访问认证网页失败"
elif [[ "$check_status" != "" ]]; then
    #尚未登录
    logger -t "【Dr.COM网页认证】" "上网登录窗尚未登录"
    # 此处如果不能正确获取ip则需要进行修改
    ip=$(ifconfig | grep inet | grep -v inet6 | grep -v 127 | grep -v 192 | awk '{print $(NF-2)}' | cut -d ':' -f2)	# 这里如果不能获取到内网ip则自己修改试试
    if [[ "$ip" == "" ]]; then
      logger -t "【Dr.COM网页认证】" "获取当前内网ip失败"
    else
      timestamp_0="$(date +%s)526"
      logger -t "【Dr.COM网页认证】" "当前内网ip：${ip}"
      timestamp_1="$(date +%s)153"
      url="http://${server}:801/eportal/?c=Portal&a=login&callback=dr${timestamp_1}&login_method=1&user_account=${user}&user_password=${password}&wlan_user_ip=${ip}&wlan_user_mac=000000000000&wlan_ac_ip=&wlan_ac_name=${wlanacname}&jsVersion=3.0&_=${timestamp_0}"
      echo $url
      curl -s "$url" \
      -H 'Connection: keep-alive' \
      -H 'User-Agent: Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/89.0.4389.90 Safari/537.36 Edg/89.0.774.57' \
      -H 'Accept: */*' \
      -H 'Referer: http://${server}/' \
      -H 'Accept-Language: zh-CN,zh;q=0.9,en;q=0.8,en-GB;q=0.7,en-US;q=0.6' \
      > ${login_out_file}
      result=`grep -Eo '"result":"[0-9]+"' ${login_out_file} | sed -r 's/"result":"([0-9]+)"/\1/g'`
      result=`[[ "$result" == 1 || "$result" == "ok" ]] && echo "登录成功" || echo "登录失败"`
      logger -t "【Dr.COM网页认证】" "上网登录窗未登录，尝试登录结果：${result} " `cat ${login_out_file}`
    fi

else
    #已经登录
    logger -t "【Dr.COM网页认证】" "上网登录窗之前已登录"
fi
logger -t "【Dr.COM网页认证】" "结束定时检测"

如果上述脚本在测试中出现断流(打游戏出现断线),可以试试笔者基于1v7w脚本修改优化的

#!/bin/bash
#重庆财经职业学院 校园网登录脚本
server=172.22.44.46    #修改成你校校园网服务器IP
user=202400000000      #修改成你的账号
password="00000000"    #修改成你的密码
wlanacname=""  # 修改成你所连接的AP的名字,我的环境不需要
html_file="/tmp/drcom_html.log"
login_out_file="/tmp/drcom_login.log"

logger -t "DrCOM" "开始检测网络"

# 1. 先 ping 认证服务器（判断是否在校园网环境）
if ! ping -c 2 -W 2 "$server" > /dev/null 2>&1; then
    logger -t "DrCOM" "无法 ping 通 $server，未连接到校园网，退出"
    echo "未连接到校园网，退出脚本"
    exit 0
fi

# 2. 再 ping 外网（判断是否已登录且网络正常）
if ping -c 1 -W 2 baidu.com > /dev/null 2>&1; then
    logger -t "DrCOM" "外网已通，无需登录"
    echo "外网已通，无需登录"
    exit 0
fi

logger -t "DrCOM" "网络受限，尝试登录"
echo "网络受限，执行登录操作"

# 3. 获取内网 IP
dev=$(ip -4 route get 1 | awk '{print $5; exit}')
ip=$(ip -4 addr show dev "$dev" | grep -o 'inet [0-9.]*' | awk '{print $2}')
if [ -z "$ip" ]; then
    logger -t "DrCOM" "无法获取内网IP"
    echo "无法获取内网IP"
    exit 1
fi

# 4. 访问认证页面确认未登录状态
curl -s "http://${server}" > ${html_file}
if grep -q "Dr.COMWebLoginID_0.htm" ${html_file}; then
    logger -t "DrCOM" "未登录状态，开始登录"
    echo "未登录，开始登录"
    url="http://${server}:801/eportal/portal/login?callback=dr1004&login_method=1&user_account=%2C0%2C${user}&user_password=${password}&wlan_user_ip=${ip}&wlan_user_ipv6=&wlan_user_mac=000000000000&wlan_ac_ip=&wlan_ac_name=${wlanacname}&jsVersion=4.2.1&terminal_type=1&lang=zh-cn&v=6528&lang=zh"
    curl -s "$url" -H "Referer: http://${server}/" > ${login_out_file}
    if grep -q '"result":1' ${login_out_file} || grep -q '"result":"ok"' ${login_out_file}; then
        logger -t "DrCOM" "登录成功"
        echo "登录成功"
    else
        logger -t "DrCOM" "登录失败"
        echo "登录失败"
    fi
else
    logger -t "DrCOM" "页面状态异常（可能已登录）但外网不通，尝试重新登录"
    echo "页面状态异常，尝试重新登录"
fi

logger -t "DrCOM" "登录检测完成"
echo "登录检测完成"

注意:部分校园网在认证成功后会立即跳转到百度等网站，导致开发者工具中的登录请求一闪而过，来不及抓包.
提供一个操作思路
以管理员身份打开记事本（右键记事本 → 以管理员身份运行）
打开 C:\Windows\System32\drivers\etc\hosts
在末尾添加一行：127.0.0.1 www.baidu.com
保存文件
抓包完成后，再次以管理员身份打开 hosts 文件，删除这一行并保存
